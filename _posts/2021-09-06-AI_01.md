# 인공지능이란


## 지능이란

> 본능적이나 자동적으로 행동하는 대신에, <red>생각</red>하고 <red>이해</red>하여 <red>행동</red>하는 능력                                

다양한 지능 요소 :             
- 연산, 감각, 추론, 학습, 운동, 감정, 사랑, 신념, 종교, 공감 등등                 


## 인공지능, AI 

> 人工知能, Artificial Intelligence       
> 인공적으로 만든 지능                
> 컴퓨터가 만들어지고 난 후부터 사람과 같은 컴퓨터도 만들 수 있을 것이다라고 생각함

인공지능에 대한 용어를 1956년 다트머스 회의에서 존 매카시가 AI용어를 제안하여 처음 사용

#### 튜링테스트 

- 지능이 있는지 없는지를 판단                    
- 지능의 조작적 정의 : 측정할 수 있는 조건으로 어떤 속성을 기술             

 
### 정의

#### 인공지능이란?

- 사람이 의식적으로 하는 행동을 컴퓨터가 할 수 있도록 하는 것         
- 사람의 생각과 관련된 활동(의사결정, 문제해결, 학습 등)을 자동화 하는 것 (벨만, 1978)                         
- 사람이 하면 더 잘할 수 있는 일을 컴퓨터가 하도록 방법을 찾는 학문 (리치, 나이트 , 1991)                       
- 지능이 요구되는 일을 할 수 있는 기계를 만드는 예술 (커즈와일, 1990)                               
- 지능적인 에이전트를 설계하는 학문 (풀, 맥워쓰, 1998)               
- 인지하고, 추론하고, 행동할 수 있도록 하는 컴퓨팅에 관련된 학문 (윌슨, 1992)                
- 인공물이 지능적인 행위를 하도록 하는 것 (닐슨, 1990)                    


### 인공지능 수준 구별

#### 약한 인공지능, Weak AI, narrow AI

$$<red>특정 문제</red>를 해결하는 지능적 행동        
사람의 지능적 행동을 <red>흉내</red> 낼 수 있는 수준                            
대부분의 인공지능 접근 방향                  
`중국인 방 사고실험`                    


#### 강한 인공지능, Strong AI
                           
$$<red>사람과 같은 인공지능</red>                    
마음을 가지고 <red>사람처럼</red> 느끼면서 지능적으로 행동하는 기계                               
추론, 문제해결, 판단, 계획, 의사소통, 자아 의식, 감정, 지혜, 양심                    
`튜링테스트`                


#### 좁은 인공지능, ANI (Aritifical Narrow Intelligence)

$$<red>특정 작업</red>에 대해서 인간 수준의 성능을 보이는 인공지능              
약한 인공지능                
일자리 개선                 


#### 범용 인공지능, AGI (Artifical General Intelligence)

$$<red>다양한 작업</red>에 대해서 인간 수준 또는 인간보다 나은 성능을 보이는 인공지능            
강한 인공지능               
일자리 위험        

#### 초인공지능, ASI (Aritifical General Intelligence)

인간 지능을 뛰어넘는 인공지능            
$$<red>자의식</red>이 있는 인공지능              
인류 위험                   


### 중국인 방 사고실험 (존설, 1980)

> 이해하지 못하고 흉내 낼 수 있어도 지능적 행동임

중국어로 쓴 질문지 전달
방 안에서 매뉴얼을 참조하여 답변에 대한 단어 조합
문 밖의 사람은 중국어를 이해하는 사람이 방 안에 있다고 생각함
흉내를 낼 뿐 이해하는 것은 아님

# 인공지능의 역사

첫번째는 1960년대에 고전적, 기본적, 정통적 방식이 1차와 2차 시기에 개발됨

탐색과 추론이 만들어지고, 지능이 요구되는 문제를 범용적 방법들에 대해 시도가 이루어짐

그러다보니 1차 때는 작은 크기의 문제는 해결할 수 있었으나

실세계의 문제를 해결하려고 보니 메모리나 시간때문에 해결하지 못함

그러다 새로운 돌파론이 생겼고, 딥러닝 기술이 크게 발전되었음

기술의 발전 속도가 비약적으로 진행중이며 새로운 기술도 발전 중

인문학에서도 시도, 활용되는 중

![image](https://user-images.githubusercontent.com/32366711/132991523-b92281ce-44b6-43e6-9cec-20dfa85c1124.png)



## 1960년대 이전

- 1946년 펜실베니아 대학, ENIAC 개발       

- 큰 기대와 여러 가지 시도, 매우 제한된 성공             
    - 여러가지 퍼즐 문제 해결 방법 개발           

- LISP[^LISP] 언어 개발 (매카시, 1958)               
    - 리스트(list) 형태의 데이터 및 지식 처리에 적합                      

- 논리(logic) 기반 지식표현 및 추론 (매카시)                

- 퍼셉트론 모델 (로젠블랏, 1958) : 하나의 뉴런

- 수단-목표 분석(means-ends) 기법 (뉴웰, 시몬, 1958)

### 수단-목표 분석 

> 해결해야 하는 문제를 <red>상태</red>로 정의               
> 현재 상태와 목표 상태간의 <red>차이</red> 계산           
> 목표 상태로 도달하기 위한 조작자를 선택 적용하는 과정 반복

작은 문제들은 쉽게 해결하지만,                  
문제의 크가기 커지게 되면 너무많은 조합이 생기게 되어            
메모리 사용량과, 시간을 만이 사용하기에 해결할 수 없어지게 됨

## 1970 ~ 1980

- 접근방법의 전환            
    - 일반적인 방법 -> <red>특정 문제 영역</red>에 효과적인 방법을 찾는 연구           
    - 이거만 잘하면 된다!                  

- <red>전문가 시스템</red>         
    - 특정 분야에 대해서 사람처럼 일할 수 있는 시스템           
    - 특정 영역의 문제에 대해서는 전문가 수준의 해답을 제공      
    - 1970년대 초반 1980년대 중반 때 상업적 성공 사례
    - MYCIN[^MYCIN], PROSPECTOR[^PROSPECTOR], DENDRAL[^DENDRAL]

- 전문가 시스템 개발 도구 개발
- 말로표현할 수없는게 많다. 데이터로부터 지식을 자동으로 뽑아내려는 시도 => 기계학습              
- Prolog 언어 개발 (콜머로어)
    - 지식의 표현과 추론을 지원하는 <red>논리</red> 기반 언어



## 1980 ~ 1990

- 신경망 모델 발전. <red>다층 퍼셉트론</red>
    - 단층 퍼셉트론은 XOR 문제를 못품
    - 퍼셉트론을 여러겹으로 놓는다면? 신경망의 르네상스
    - XOR 문제와 같은 비선형 분할 문제 해결 가능
    - $$<red>오차 역전파</red> 알고리즘 사용

- 퍼지이론 (fuzzy theory)
    - 언어적인 애매한 정보를 정량화하여 표현          

- 진화 연산(evolutionary computation)
    - 진화 개념을 문제 해결에 도입
    - 유전자 알고리즘, 진화 프로그래밍 등

- 확률적 그래프 모델 (Probabilistic graphical model)
    - 그래프 이론과 확률론 결합
    - 컴퓨터 비전, 로보틱스, 자연어 처리, 진단 등에 활용

- 서포트 벡터 머신 (SVM, Support Vector Machine)
    - 이때까지 가장 좋은 분류모델
    - 데이터 사이즈가 작으면 좋은 성능을 보이지만, 사이즈가 커지면 성능이 저하됨

- 수치계산 위주의 비기호적 인공지능(subsymbolic AI) 기술의 발전

- 자연어 처리



## 2000년 이후

딥러닝 기술이 특히 발전하면서 상업적 기능 발전              
일상에서도 활용하게 됨                

- 에이전트
    - 위임받은 일을 자율적으로 처리

- 시맨틱 웹
    - 컴퓨터가 웹 상의 자원을 이해하여 처리

- 기계학습          
- 데이터 마이닝          
- 딥러닝          


# 연구 지능의 요소 기술

## 탐색 (Search)

> 문제의 답이 될 수 있는 것들의 집합을 <red>공간</red>으로 간주하고, 문제에 대한 <red>최적의 해</red>를 찾기 위해 <red>공간</red>을 <red>체계적으로 찾아보는 것</red>           

#### 무정보 탐색

- 너비우선 탐색, 깊이우선 탑색
- 실제 인공지능은 대상이 너무 많음 => 시간, 메모리를 너무 많이차지하게 됨. 비효율적

#### 휴리스틱 탐색

- 언덕오르기 탐색, 최선 운선탐색, 빔탐색, A*알고리즘

#### 게임트리 탐색

- mini-max 알고리즘, $\alpha - \beta$가지치기(pruning), 몬테카를로 트리 탐색


## 지식표현 (knowledge representation)

> 문제 해결에 이용하거나, 심층적 추론을 할 수 있도록 지식을 효과적으로 표현하는 방법

- IF-THEN 규칙           
- 프레임           
- 의미망           
- 지식 그래프           
- 논리(logic) : 명제논리, 술어논리           
- 스크립트           
- 온톨로지 기술 언어 : RDF, OWL           
- 불확실한 지식 표현 : 확신도, 확률기반 표현, 퍼지이론           
- 확률 그래프 모델              
- 함수 기반 지식표현         


## 추론 (inference)

> 가정이나 전제로부터 결론을 이끌어내는 것

- 규칙 기반 시스템의 추론                 
    - 전향추론(forward inference), 후향추론(backward inference)               

- 확률 모델의 추론        
    - 관심 대상의 확률 또는 확률분포를 결정하는 것
    - 베이즈 정리 및 주변화 이용

![image](https://user-images.githubusercontent.com/32366711/132995035-320e5c4d-5c00-4665-a0ec-95e11f7eb621.png){: width="500"}


## 기계학습

> $$<red>경험</red>을 통해서 나중에 유사하거나 같은 일을 더 효율적으로 처리할 수 있도록 시스템의 구조나 파라미터를 바꾸는 것

- 지도학습       
    - 입력과 대응하는 출력을 데이터로 제공하고 대응 관계의 함수 찾기
    - 정답을 제공하고 정답률 확인

- 비지도학습
    - 데이터만 주어진 상태에서 유사한 것들을 서로 묶어 군집을 찾거나 확률분포 표현        
    - 정답 없이 확인

- 강화학습
    - 상황 별 행동에 따른 시스템의 보상 값(reward value)만을 이용하여, 시스템에 대한 바랍직한 행동 정책 찾기


## 계획수립

> 현재 상태에서 목표하는 상태에 도달하기 위해 수행해야 할 일련의 행동순서를 결정하는 것

- 작업 수행 절차 계획             
- 로봇의 움직임 계획           


# 주요 응용 분야

## 전문가 시스템

> $$<red>특정 문제</red>에 <red>전문가</red>수준의 해법을 제공                   
> 간단한 제어시스템에서부터 복잡한 계산과 추론을 요구하는 의료진단, 고장 진단, 추천 시스템                         
> $$<red>신속하게 변화</red>해야하는 곳에서 전문가 시스템이 좋음                      

- 작업 중요도가 높은 분야의 경우 추천 정보로 활용                  
    - 최종 결정은 현장 작업자 담당               
    - 원자력 발전소 항공우주 분야 등            

- 지식 표현과 추론 부분 분리하여 구성            
    - 지식만 변경하면 변화하는 환경에 쉽게 대응
           
- 규칙기반 시스템을 통한 구현              

## 데이터 마이닝

> 실제 대규모 데이터에서 암묵적인, 이전에 알려지지 않은,                   
> 잠재적으로 유용할 것 같은 정보를 추출하는 체계적인 과정               
> 기계학습 통계학 기법 적용

- 연관 규칙, 분류 패턴, 군집화 패턴, 텍스트 마이닝, 그래프 마이닝, 추천, 시각화

## 패턴인식

> 데이터에 있는 패턴이나 규칙성을 찾는 것
> 문자, 음성, 영상 인식        
> 텍스트 패턴 인식, 섹서 신호 인식, 이상치 탐색

## 자연어 처리

> 언어를 처리하는 분야
> 사람이 사용하는 일반 언어로 작성된 문서를 처리하고 이해하는 분야
> 이후에 어떤 내용이 나올 것인가

- 형태소 분석, 구문분석, 품사 태깅, 의미분석        
- 언어모델, 주제어 추출, 객체명 인식        
- 문서 요약                
- 기계번역        
- 질의 응답         

## 컴퓨터 비전

> 컴퓨터를 이용하여 시각 기능을 갖는 기계장치를 만들려는 분야             

- 영상처리         
    - 원 영상을 사용 목적에 맞게 가공                                                    
           
## 음성 인식

> 사람의 음성 언어를 컴퓨터가 해석해 그 내용을 문자 데이터로 전환하는 처리

- 다양한 응용 서비스           
    - 음성인식 인터페이스, 인공지능 스피커                             

## 로보틱스

> 로봇에 관련된 기술 분야로서 기계공학, 센서공학, 마이크로 일렉트로닉스, 인공지능 기술 등을 종합적으로 활용

- 지능 로봇
    - 인공지능 기술을 활용하는 로봇

## 에이전트  

> 사용자로부터 위임받은 일을 자율적으로 수행하는 시스템

- 소프트웨어 에이전트                                                
    - 채팅 봇                                       
- 물리적 에이전트                                       
    - 로봇                              
- 반응형 에이전트               
    - 단순히 입력에 대해서 정해진 반응                                        
- 숙고형 에이전트                             
    - 자신의 지식을 활용하여, 목표를 달성하기 위한 계획을 수립하고 수행                     
- 학습에이전트                 
    - 경험이 누적됨에 따라 점점 똑똑해지도록 학습

    - 비구조화된 데이터 분석
    - 의료, 금융, 유통 등 다양한 분야에 활용

    - 한국어 질의응답 시스템

- Project Debater

- 자율주행 자동차
    - 인지 기술, 판단 기술, 제어 기술, 센서 기술, 통신 기술

        - 3 : 조건부 자율주행, 특정 조건에서 자율주행
        - 4 : 고도 자동화
        - 5 : 완전 자동화

# 인공지능의 영향

## 인공지능 기술 도입

> 자동화 => 생산성 향상

- 일자리 문제         
    - 블루칼라[^blue] 일자리 축소
    - 화이트 칼라[^white] 일자리 축소
    - 노동력 잉여 발생

- 사회적 문제 초래
    - 고용 및 일자리, 기회의 불평등, 소극의 양극화 등

## 인공지능의 윤리

- 살상용 자율무기

- 자율주행 자동차의 돌발 상황에 대한 프로그래밍

- 딥페이크 : 인공지능 기술을 이용한 위조데이터 생성

### 편향성

- 데이터의 편향성                  
    - 데이터 부족에 따른 과적합                
    - 데이터 편향에 따른 대상 집단 대표성 상실         
    - 대상 집단의 문화적 편향에 따른 데이터의 편향                   

- 알고리즘의 편향성              
    - 데이터 처리 알고리즘의 왜곡              
    - 학습 알고리즘의 불완전성             

- 개발자의 편향성               
    - 문화적 고정관념                   


## 인공지능의 민주화

> 누구나 고급 인공지능 기술에 접근하고 사용할 수 있는 것                      
> 사용자 수준에 무관하게 필요 기술 사용                       
> 지역 국가 무관하게 필요 기술 사용                    

## 특이점

> 기술의 수준이 어느 한순간 기하급수적으로 증가하는 시점
> 초인간 인공지능 개발은 인류 종말의 시점일 것

#### 인공지능의 특이점

- 인공지능이 인간의 지능보다 더 진보하게 되는 시점

[^PROSPECTOR]: 광물탐사 데이터 분석
